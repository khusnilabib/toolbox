    11тЖТ    11тЖТ## 1. Purpose
    12тЖТ    12тЖТ
    13тЖТ    13тЖТThis Design System document defines the visual language of [PROJECT_NAME]: design tokens (colors, typography, spacing, radius, shadows, motion), component patterns, theming (light/dark mode), and accessibility standards. It operationalizes LOCK-10 (developer-first minimalism inspired by Vercel-like aesthetic) and the Engineering Constitution articles that govern design: EC-06 (accessibility first) and EC-10 (design system governance).
    14тЖТ    14тЖТ
    15тЖТ    15тЖТA design system is the single source of truth for visual decisions. Without one, every engineer and designer makes slightly different choices тАФ button radii drift from 6px to 8px to 4px; grays shift from `#666` to `#555` to `#777`; spacing varies from 12px to 16px to 14px. Individually trivial, collectively these inconsistencies make a platform feel unprofessional. The design system prevents this by encoding every visual decision as a token, and every token is consumed via Tailwind classes.
    16тЖТ    16тЖТ
    17тЖТ    17тЖТThis document is enforced via Tailwind configuration (tokens mapped to Tailwind theme), shadcn/ui component library (consistent primitives), ESLint rules (no inline styles, no hardcoded colors), and code review. Per EC-10, no page may introduce ad hoc UI patterns тАФ every visual element originates from this system.

    40тЖТ    40тЖТ## 3. Architectural Decisions
    41тЖТ    41тЖТ
    42тЖТ    42тЖТ### AD-01 тАФ Token-Driven Theming via CSS Custom Properties
    43тЖТ    43тЖТ
    44тЖТ    44тЖТ**Context.** Hardcoded colors in components break dark mode and prevent theming. CSS-in-JS solutions add runtime overhead. A token-driven approach using CSS custom properties enables theme switching without re-renders.
    45тЖТ    45тЖТ
    46тЖТ    46тЖТ**Decision.** Design tokens are defined as CSS custom properties in `globals.css` at the `:root` level (light theme) and `.dark` class (dark theme). Tailwind theme extension maps these tokens to Tailwind classes (e.g., `bg-background` тЖТ `background-color: var(--background)`).
    47тЖТ    47тЖТ
    48тЖТ    48тЖТ```css
    49тЖТ    49тЖТ:root {
    50тЖТ    50тЖТ  --background: 0 0% 100%;           /* white */
    51тЖТ    51тЖТ  --foreground: 0 0% 3.9%;           /* near black */
    52тЖТ    52тЖТ  --primary: 0 0% 9%;                /* primary action */
    53тЖТ    53тЖТ  --primary-foreground: 0 0% 98%;
    54тЖТ    54тЖТ  /* ... more tokens */
    55тЖТ    55тЖТ}
    56тЖТ    56тЖТ
    57тЖТ    57тЖТ.dark {
    58тЖТ    58тЖТ  --background: 0 0% 3.9%;
    59тЖТ    59тЖТ  --foreground: 0 0% 98%;
    60тЖТ    60тЖТ  --primary: 0 0% 98%;
    61тЖТ    61тЖТ  --primary-foreground: 0 0% 9%;
    62тЖТ    62тЖТ  /* ... more tokens */
    63тЖТ    63тЖТ}
    64тЖТ    64тЖТ```
    65тЖТ    65тЖТ
    66тЖТ    66тЖТTailwind config:
    67тЖТ    67тЖТ```typescript
    68тЖТ    68тЖТmodule.exports = {
    69тЖТ    69тЖТ  theme: {
    70тЖТ    70тЖТ    extend: {
    71тЖТ    71тЖТ      colors: {
    72тЖТ    72тЖТ        background: 'hsl(var(--background))',
    73тЖТ    73тЖТ        foreground: 'hsl(var(--foreground))',
    74тЖТ    74тЖТ        primary: { DEFAULT: 'hsl(var(--primary))', foreground: 'hsl(var(--primary-foreground))' },
    75тЖТ    75тЖТ        // ... more
    76тЖТ    76тЖТ      },
    77тЖТ    77тЖТ    },
    78тЖТ    78тЖТ  },
    79тЖТ    79тЖТ};
    80тЖТ    80тЖТ```
    81тЖТ    81тЖТ
    82тЖТ    82тЖТ**Consequences.**
    83тЖТ    83тЖТ- тЬЕ Theme switching via class toggle on `<html>` тАФ no re-renders.
    84тЖТ    84тЖТ- тЬЕ Consistent token usage across all components.
    85тЖТ    85тЖТ- тЬЕ Dark mode supported from day 1 (LOCK-10).
    86тЖТ    86тЖТ- тЪая╕П Slight indirection (token тЖТ CSS var тЖТ Tailwind class); justified by flexibility.
    87тЖТ    87тЖТ
    88тЖТ    88тЖТ**Implements:** LOCK-10 (Dark/Light mode ready), EC-10 (Design System Governance).

    90тЖТ    90тЖТ### AD-02 тАФ Monochrome Palette with Single Accent
    91тЖТ    91тЖТ
    92тЖТ    92тЖТ**Context.** LOCK-10 mandates developer-first minimalism with a monochrome palette. Adding multiple accent colors fragments the visual language.
    93тЖТ    93тЖТ
    94тЖТ    94тЖТ**Decision.** Palette:
    95тЖТ    95тЖТ- **Background and foreground:** pure grayscale (white to near-black).
    96тЖТ    96тЖТ- **Primary actions:** near-black in light mode, near-white in dark mode (inverted).
    97тЖТ    97тЖТ- **Accent color:** single accent reserved for highlighting (links, focused elements, badges). The accent is configurable via token; default is a subtle blue (`hsl(221, 83%, 53%)`) for familiarity, but can be re-themed without code changes.
    98тЖТ    98тЖТ- **Semantic colors:** success (green), warning (yellow), destructive (red), info (blue). Used sparingly for status indicators, not decoration.
    99тЖТ    99тЖТ
   100тЖТ   100тЖТAll other colors are grayscale. No gradients, no decorative color.
   101тЖТ   101тЖТ
   102тЖТ   102тЖТ**Consequences.**
   103тЖТ   103тЖТ- тЬЕ Consistent, professional aesthetic.
   104тЖТ   104тЖТ- тЬЕ High contrast (WCAG AAA where feasible тАФ EC-06).
   105тЖТ   105тЖТ- тЬЕ Color-blind friendly (semantic meaning conveyed via color + icon, not color alone).
   106тЖТ   106тЖТ
   107тЖТ   107тЖТ**Implements:** LOCK-10, EC-06.

   109тЖТ   109тЖТ### AD-03 тАФ Typography Scale
   110тЖТ   110тЖТ
   111тЖТ   111тЖТ**Context.** Without a defined type scale, font sizes drift (14px, 15px, 16px, 17px all appearing). A modular scale ensures hierarchy and rhythm.
   112тЖТ   112тЖТ
   113тЖТ   113тЖТ**Decision.** Typography scale (rem-based, 1rem = 16px):
   114тЖТ   114тЖТ
   115тЖТ   115тЖТ| Token | Size | Weight | Use |
   116тЖТ   116тЖТ|-------|------|--------|-----|
   117тЖТ   117тЖТ| `text-xs` | 0.75rem (12px) | 400 | Captions, badges, small print |
   118тЖТ   118тЖТ| `text-sm` | 0.875rem (14px) | 400 | Secondary text, labels, table cells |
   119тЖТ   119тЖТ| `text-base` | 1rem (16px) | 400 | Body text, default |
   120тЖТ   120тЖТ| `text-lg` | 1.125rem (18px) | 500 | Section headings, emphasized text |
   121тЖТ   121тЖТ| `text-xl` | 1.25rem (20px) | 600 | Page subheadings |
   122тЖТ   122тЖТ| `text-2xl` | 1.5rem (24px) | 600 | Page headings |
   123тЖТ   123тЖТ| `text-3xl` | 1.875rem (30px) | 700 | Hero headings |
   124тЖТ   124тЖТ| `text-4xl` | 2.25rem (36px) | 700 | Large hero, marketing |
   125тЖТ   125тЖТ| `text-5xl` | 3rem (48px) | 700 | Marketing only |
   126тЖТ   126тЖТ
   127тЖТ   127тЖТ**Font families:**
   128тЖТ   128тЖТ- **Sans-serif (default):** `Inter` (loaded via `next/font/google`). Used for UI, body text.
   129тЖТ   129тЖТ- **Mono:** `JetBrains Mono` (loaded via `next/font/google`). Used for code blocks, technical identifiers.
   130тЖТ   130тЖТ- **No serif fonts** in Phase 1. May add for editorial content (articles) in Phase 2.
   131тЖТ   131тЖТ
   132тЖТ   132тЖТ**Line heights:**
   133тЖТ   133тЖТ- Body: 1.5
   134тЖТ   134тЖТ- Headings: 1.2
   135тЖТ   135тЖТ- Code: 1.6
   136тЖТ   136тЖТ
   137тЖТ   137тЖТ**Letter spacing:**
   138тЖТ   138тЖТ- Body: normal (0)
   139тЖТ   139тЖТ- Headings: -0.01em (slightly tighter for large text)
   140тЖТ   140тЖТ- Small text: 0.01em (slightly looser for readability)
   141тЖТ   141тЖТ
   142тЖТ   142тЖТ**Implements:** LOCK-10 (premium feel via precise typography).

   144тЖТ   144тЖТ### AD-04 тАФ Spacing Scale
   145тЖТ   145тЖТ
   146тЖТ   146тЖТ**Context.** Inconsistent spacing (12px, 14px, 16px, 18px padding) breaks visual rhythm. Tailwind's spacing scale (4px base) provides consistency.
   147тЖТ   147тЖТ
   148тЖТ   148тЖТ**Decision.** Use Tailwind's default spacing scale:
   149тЖТ   149тЖТ
   150тЖТ   150тЖТ| Token | Size | Use |
   151тЖТ   151тЖТ|-------|------|-----|
   152тЖТ   152тЖТ| `0` | 0 | No spacing |
   153тЖТ   153тЖТ| `1` | 4px | Tight inline spacing |
   154тЖТ   154тЖТ| `2` | 8px | Compact element padding |
   155тЖТ   155тЖТ| `3` | 12px | Small element padding |
   156тЖТ   156тЖТ| `4` | 16px | Default element padding |
   157тЖТ   157тЖТ| `6` | 24px | Section internal spacing |
   158тЖТ   158тЖТ| `8` | 32px | Section spacing |
   159тЖТ   159тЖТ| `12` | 48px | Page section spacing |
   160тЖТ   160тЖТ| `16` | 64px | Large page section spacing |
   161тЖТ   161тЖТ| `24` | 96px | Hero spacing |
   162тЖТ   162тЖТ
   163тЖТ   163тЖТ**Rules:**
   164тЖТ   164тЖТ- Use only these values; no `p-[13px]` arbitrary values.
   165тЖТ   165тЖТ- Spacing between related elements: `2` or `3` (8-12px).
   166тЖТ   166тЖТ- Spacing within a section: `4` or `6` (16-24px).
   167тЖТ   167тЖТ- Spacing between sections: `8` or `12` (32-48px).
   168тЖТ   168тЖТ- Page-level rhythm: multiples of 8px.
   169тЖТ   169тЖТ
   170тЖТ   170тЖТ**Implements:** LOCK-10 (spacious layout), EC-10 (consistent spacing).

   173тЖТ   173тЖТ
   174тЖТ   174тЖТ**Context.** Inconsistent corner radii (4px, 6px, 8px, 12px) look unprofessional.
   175тЖТ   175тЖТ
   176тЖТ   176тЖТ**Decision.** Radius scale:
   177тЖТ   177тЖТ
   178тЖТ   178тЖТ| Token | Radius | Use |
   179тЖТ   179тЖТ|-------|--------|-----|

   185тЖТ   185тЖТ| `rounded-full` | 9999px | Pills, avatars, circular buttons |
   186тЖТ   186тЖТ
   187тЖТ   187тЖТ**Rules:**
   188тЖТ   188тЖТ- Default radius: `rounded-md` (6px).
   189тЖТ   189тЖТ- Cards: `rounded-lg` (8px) or `rounded-xl` (12px) for emphasis.
   190тЖТ   190тЖТ- Inputs and buttons: `rounded-md` (6px).

   195тЖТ   195тЖТ### AD-06 тАФ Shadow Scale
   196тЖТ   196тЖТ
   197тЖТ   197тЖТ**Context.** Shadows convey depth. Without a scale, shadows vary wildly.
   198тЖТ   198тЖТ
   199тЖТ   199тЖТ**Decision.** Shadow scale (subtle, developer-first aesthetic):
   200тЖТ   200тЖТ
   201тЖТ   201тЖТ| Token | Shadow | Use |

   212тЖТ   212тЖТ- Use `shadow-md` for cards with hover interaction.
   213тЖТ   213тЖТ- Use `shadow-lg` for modals and popovers.
   214тЖТ   214тЖТ- Dark mode: shadows are less visible; rely more on border for depth.

   216тЖТ   216тЖТ**Implements:** LOCK-10 (minimal animation, premium feel via precise elevation).

   218тЖТ   218тЖТ### AD-07 тАФ Motion and Animation Standards
   219тЖТ   219тЖТ
   220тЖТ   220тЖТ**Context.** LOCK-10 mandates minimal animation. Excessive motion degrades the developer-first aesthetic and can cause accessibility issues (motion sensitivity).
   221тЖТ   221тЖТ
   222тЖТ   222тЖТ**Decision.** Motion standards:
   223тЖТ   223тЖТ
   224тЖТ   224тЖТ**Allowed animations (state-change only):**
   225тЖТ   225тЖТ- Hover transitions: 150ms ease-out.
   226тЖТ   226тЖТ- Focus transitions: 100ms ease-out.
   227тЖТ   227тЖТ- Modal/dialog open: 200ms ease-out.
   228тЖТ   228тЖТ- Modal/dialog close: 150ms ease-in.
   229тЖТ   229тЖТ- Toast notifications: 200ms ease-out.
   230тЖТ   230тЖТ
   231тЖТ   231тЖТ**Forbidden animations:**
   232тЖТ   232тЖТ- Decorative animations (pulsing, bouncing, rotating for non-loading states).
   233тЖТ   233тЖТ- Auto-playing carousels.
   234тЖТ   234тЖТ- Parallax scrolling.
   235тЖТ   235тЖТ- Animated illustrations.
   236тЖТ   236тЖТ- Page transition animations (instant page changes preferred).
   237тЖТ   237тЖТ
   238тЖТ   238тЖТ**Loading states:**
   239тЖТ   239тЖТ- Skeleton loaders with subtle pulse (1.5s ease-in-out infinite).
   240тЖТ   240тЖТ- Progress bars for long operations.
   241тЖТ   241тЖТ- No spinner-only loading states (always skeleton or progress).
   242тЖТ   242тЖТ
   243тЖТ   243тЖТ**Reduced motion:**
   244тЖТ   244тЖТ- All animations respect `prefers-reduced-motion: reduce` (EC-06).
   245тЖТ   245тЖТ- When reduced motion is preferred, animations are instant or replaced with opacity changes.
   246тЖТ   246тЖТ
   247тЖТ   247тЖТ```css
   248тЖТ   248тЖТ@media (prefers-reduced-motion: reduce) {
   249тЖТ   249тЖТ  *, *::before, *::after {
   250тЖТ   250тЖТ    animation-duration: 0.01ms !important;
   251тЖТ   251тЖТ    transition-duration: 0.01ms !important;
   252тЖТ   252тЖТ  }
   253тЖТ   253тЖТ}
   254тЖТ   254тЖТ```
   255тЖТ   255тЖТ
   256тЖТ   256тЖТ**Implements:** LOCK-10 (minimal animation), EC-06 (reduced motion compatibility).

   316тЖТ   316тЖТ
   317тЖТ   317тЖТ**Keyboard navigation:**
   318тЖТ   318тЖТ- All interactive elements operable via keyboard.
   319тЖТ   319тЖТ- Logical tab order (follows visual order).
   320тЖТ   320тЖТ- Visible focus indicator (`focus-visible:` Tailwind classes with `outline-2 outline-offset-2 outline-primary`).
   321тЖТ   321тЖТ- No keyboard traps.
   322тЖТ   322тЖТ- Skip-to-content link at top of every page.
   323тЖТ   323тЖТ
   324тЖТ   324тЖТ**Screen reader support:**
   325тЖТ   325тЖТ- Semantic HTML (`<button>`, `<nav>`, `<main>`, `<article>`, `<section>`, `<aside>`, `<header>`, `<footer>`).
   326тЖТ   326тЖТ- ARIA labels only when semantic HTML insufficient.
   327тЖТ   327тЖТ- `aria-label` on icon-only buttons.
   328тЖТ   328тЖТ- `aria-live="polite"` on dynamic content (toasts, alerts).
   329тЖТ   329тЖТ- `aria-busy` on loading elements.
   330тЖТ   330тЖТ- Hidden content via `aria-hidden="true"` (not `display: none` if it should be screen-reader accessible).
   331тЖТ   331тЖТ
   332тЖТ   332тЖТ**Forms:**
   333тЖТ   333тЖТ- Every input has associated `<label>`.
   334тЖТ   334тЖТ- Error messages via `aria-describedby` linking to input.
   335тЖТ   335тЖТ- Required fields marked with `aria-required="true"` and visible indicator.
   336тЖТ   336тЖТ- Validation errors announced via `aria-live`.
   337тЖТ   337тЖТ
   338тЖТ   338тЖТ**Reduced motion:**
   339тЖТ   339тЖТ- Per AD-07. `prefers-reduced-motion` respected.
   340тЖТ   340тЖТ
   341тЖТ   341тЖТ**Touch targets:**
   342тЖТ   342тЖТ- Minimum 44x44px on touch devices.
   343тЖТ   343тЖТ- Adequate spacing between interactive elements.
   344тЖТ   344тЖТ
   345тЖТ   345тЖТ**Images:**
   346тЖТ   346тЖТ- All images have `alt` attribute.
   347тЖТ   347тЖТ- Decorative images: `alt=""` (empty).
   348тЖТ   348тЖТ- Complex images: `aria-describedby` linking to longer description.
   349тЖТ   349тЖТ
   350тЖТ   350тЖТ**Implements:** EC-06 (Accessibility First).
   351тЖТ   351тЖТ
   352тЖТ   352тЖТ### AD-11 тАФ Dark Mode Strategy
   353тЖТ   353тЖТ
   354тЖТ   354тЖТ**Context.** LOCK-10 mandates dark mode ready from day 1. The strategy must be token-driven (no hardcoded colors) and toggleable.
   355тЖТ   355тЖТ
   356тЖТ   356тЖТ**Decision.** Dark mode strategy:
   357тЖТ   357тЖТ- **Token-driven:** All colors via CSS custom properties (AD-01).
   358тЖТ   358тЖТ- **Class-based theming:** `.dark` class on `<html>` element toggles dark mode.
   359тЖТ   359тЖТ- **User preference:** Stored in cookie (server-readable for SSR) and localStorage.
   360тЖТ   360тЖТ- **System preference:** Default to `prefers-color-scheme` if no user preference.
   361тЖТ   361тЖТ- **No flash of incorrect theme:** Server-rendered `<html>` class based on cookie.
   362тЖТ   362тЖТ
   363тЖТ   363тЖТ**Implementation:**
   364тЖТ   364тЖТ```typescript
   365тЖТ   365тЖТ// src/app/layout.tsx
   366тЖТ   366тЖТexport async function generateMetadata() {
   367тЖТ   367тЖТ  // ...
   368тЖТ   368тЖТ}
   369тЖТ   369тЖТ
   370тЖТ   370тЖТexport default function RootLayout({ children }: { children: React.ReactNode }) {
   371тЖТ   371тЖТ  return (
   372тЖТ   372тЖТ    <html lang="en" suppressHydrationWarning>
   373тЖТ   373тЖТ      <head>
   374тЖТ   374тЖТ        <script
   375тЖТ   375тЖТ          dangerouslySetInnerHTML={{
   376тЖТ   376тЖТ            __html: `
   377тЖТ   377тЖТ              (function() {
   378тЖТ   378тЖТ                const theme = document.cookie.match(/theme=(dark|light)/)?.[1]
   379тЖТ   379тЖТ                  || (window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light');
   380тЖТ   380тЖТ                document.documentElement.classList.toggle('dark', theme === 'dark');
   381тЖТ   381тЖТ              })();
   382тЖТ   382тЖТ            `,
   383тЖТ   383тЖТ          }}
   384тЖТ   384тЖТ        />
   385тЖТ   385тЖТ      </head>
   386тЖТ   386тЖТ      <body>{children}</body>
   387тЖТ   387тЖТ    </html>
   388тЖТ   388тЖТ  );
   389тЖТ   389тЖТ}
   390тЖТ   390тЖТ```
   391тЖТ   391тЖТ
   392тЖТ   392тЖТ**Implements:** LOCK-10 (Dark/Light mode ready).
   393тЖТ   393тЖТ
   394тЖТ   394тЖТ### AD-12 тАФ Z-Index Scale
   395тЖТ   395тЖТ
   396тЖТ   396тЖТ**Context.** Without a z-index scale, values drift and stacking becomes unpredictable.
   397тЖТ   397тЖТ
   398тЖТ   398тЖТ**Decision.** Z-index scale:
   399тЖТ   399тЖТ
   400тЖТ   400тЖТ| Token | Value | Use |
   401тЖТ   401тЖТ|-------|-------|-----|
   402тЖТ   402тЖТ| `z-0` | 0 | Default |
   403тЖТ   403тЖТ| `z-10` | 10 | Sticky headers, dropdowns |
   404тЖТ   404тЖТ| `z-20` | 20 | Sticky sidebars |
   405тЖТ   405тЖТ| `z-30` | 30 | Modals |
   406тЖТ   406тЖТ| `z-40` | 40 | Popovers, tooltips |
   407тЖТ   407тЖТ| `z-50` | 50 | Toast notifications |
   408тЖТ   408тЖТ| `z-[100]` | 100 | Critical overlays (error boundaries) |
   409тЖТ   409тЖТ
   410тЖТ   410тЖТ**Rules:**
   411тЖТ   411тЖТ- Use only these values.
   412тЖТ   412тЖТ- Never use `z-[9999]` or other arbitrary values.
   413тЖТ   413тЖТ- Higher z-index = more "on top"; respect the hierarchy.
   414тЖТ   414тЖТ
   415тЖТ   415тЖТ**Implements:** EC-10 (consistent stacking).
   416тЖТ   416тЖТ
   417тЖТ   417тЖТ## 4. Design Principles
   418тЖТ   418тЖТ
   419тЖТ   419тЖТ### P1 тАФ Token-Driven, Always
   420тЖТ   420тЖТEvery color, spacing, radius, shadow, font size comes from a token. No hardcoded values. This is the foundation of dark mode and theming.
   421тЖТ   421тЖТ
   422тЖТ   422тЖТ### P2 тАФ Minimal by Default
   423тЖТ   423тЖТStart with the minimum viable visual treatment. Add decoration only when it improves usability. LOCK-10 mandates this.
   424тЖТ   424тЖТ
   425тЖТ   425тЖТ### P3 тАФ Hierarchy Through Contrast, Not Decoration
   426тЖТ   426тЖТVisual hierarchy via typography size/weight, spacing, and color contrast тАФ not via decorative elements (borders, gradients, shadows).
   427тЖТ   427тЖТ
   428тЖТ   428тЖТ### P4 тАФ Accessible by Construction
   429тЖТ   429тЖТAccessibility is built into tokens (contrast ratios) and components (semantic HTML, ARIA). Not bolted on later.
   430тЖТ   430тЖТ
   431тЖТ   431тЖТ### P5 тАФ Consistent Across the Ecosystem
   432тЖТ   432тЖТEvery tool page, every admin page, every marketing page uses the same tokens and components. The user feels they're in one product, not a patchwork.
   433тЖТ   433тЖТ
   434тЖТ   434тЖТ### P6 тАФ Performance-Conscious
   435тЖТ   435тЖТMinimal CSS, no runtime CSS-in-JS, fonts loaded via `next/font` for optimization. Design choices must not bloat the bundle.
   436тЖТ   436тЖТ

   721тЖТ   721тЖТ## 8. Standards
   722тЖТ   722тЖТ
   723тЖТ   723тЖТ### 8.1 Token Usage Standards
   724тЖТ   724тЖТ- All colors via tokens (`bg-background`, `text-foreground`, etc.).
   725тЖТ   725тЖТ- No hardcoded hex values.
   726тЖТ   726тЖТ- No `style={{ color: '#fff' }}` inline styles.
   727тЖТ   727тЖТ
   728тЖТ   728тЖТ### 8.2 Component Usage Standards
   729тЖТ   729тЖТ- Use design system components from `@packages/ui`.
   730тЖТ   730тЖТ- Compose into higher-level components in `src/shared/components/`.
   731тЖТ   731тЖТ- Don't reinvent primitives (Button, Input, Card, etc.).
   732тЖТ   732тЖТ
   733тЖТ   733тЖТ### 8.3 Dark Mode Standards
   734тЖТ   734тЖТ- Every component works in light and dark mode.
   735тЖТ   735тЖТ- Test both modes during development.
   736тЖТ   736тЖТ- No dark-mode-specific code paths; token switching handles it.
   737тЖТ   737тЖТ
   738тЖТ   738тЖТ### 8.4 Accessibility Standards
   739тЖТ   739тЖТ- Per AD-10. WCAG AA conformance mandatory.
   740тЖТ   740тЖТ- Verified via Lighthouse CI (тЙе95 accessibility score).
   741тЖТ   741тЖТ
   742тЖТ   742тЖТ### 8.5 Animation Standards
   743тЖТ   743тЖТ- Per AD-07. Minimal animation.
   744тЖТ   744тЖТ- Reduced motion respected.
   745тЖТ   745тЖТ
   746тЖТ   746тЖТ### 8.6 Performance Standards
   747тЖТ   747тЖТ- No runtime CSS-in-JS.
   748тЖТ   748тЖТ- Fonts via `next/font`.
   749тЖТ   749тЖТ- Minimal custom CSS (only `globals.css`).
   750тЖТ   750тЖТ
   751тЖТ   751тЖТ## 9. Best Practices
   752тЖТ   752тЖТ
   753тЖТ   753тЖТ### 9.1 When Choosing Colors
   754тЖТ   754тЖТ1. Use tokens, never hex values.
   755тЖТ   755тЖТ2. Default to muted/foreground for text.
   756тЖТ   756тЖТ3. Use primary for actions, accent for highlights.
   757тЖТ   757тЖТ4. Use semantic colors (success/warning/destructive) only for status.
   758тЖТ   758тЖТ5. Verify contrast in both light and dark mode.
   759тЖТ   759тЖТ
   760тЖТ   760тЖТ### 9.2 When Spacing Elements
   761тЖТ   761тЖТ1. Use Tailwind spacing scale (4, 8, 12, 16, 24, 32, 48, 64).
   762тЖТ   762тЖТ2. Within a section: 16-24px (`gap-4` or `gap-6`).
   763тЖТ   763тЖТ3. Between sections: 32-48px (`py-8` or `py-12`).
   764тЖТ   764тЖТ4. Page padding: 16px mobile, 24-32px desktop.
   765тЖТ   765тЖТ
   766тЖТ   766тЖТ### 9.3 When Adding a Component
   767тЖТ   767тЖТ1. Check `@packages/ui` for existing (EC-03).
   768тЖТ   768тЖТ2. Check `src/shared/components/` for app-specific composite.
   769тЖТ   769тЖТ3. If new: place in appropriate location per `07_FolderStructure`.
   770тЖТ   770тЖТ4. Use design tokens, not hardcoded values.
   771тЖТ   771тЖТ5. Verify dark mode and accessibility.
   772тЖТ   772тЖТ
   773тЖТ   773тЖТ### 9.4 When Animating
   774тЖТ   774тЖТ1. Default to no animation.
   775тЖТ   775тЖТ2. Add animation only for state changes (hover, focus, open/close).
   776тЖТ   776тЖТ3. Keep duration тЙд200ms.
   777тЖТ   777тЖТ4. Respect `prefers-reduced-motion`.
   778тЖТ   778тЖТ5. No decorative animations.
   779тЖТ   779тЖТ
   780тЖТ   780тЖТ### 9.5 When Theming
   781тЖТ   781тЖТ1. All theming via tokens.
   782тЖТ   782тЖТ2. To change theme: modify CSS custom properties in `globals.css`.
   783тЖТ   783тЖТ3. Dark mode: toggle `.dark` class on `<html>`.
   784тЖТ   784тЖТ4. Test both modes.
   785тЖТ   785тЖТ
