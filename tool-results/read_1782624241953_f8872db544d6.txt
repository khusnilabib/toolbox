    11→## 1. Purpose
    12→
    13→This Design System document defines the visual language of [PROJECT_NAME]: design tokens (colors, typography, spacing, radius, shadows, motion), component patterns, theming (light/dark mode), and accessibility standards. It operationalizes LOCK-10 (developer-first minimalism inspired by Vercel-like aesthetic) and the Engineering Constitution articles that govern design: EC-06 (accessibility first) and EC-10 (design system governance).
    14→
    15→A design system is the single source of truth for visual decisions. Without one, every engineer and designer makes slightly different choices — button radii drift from 6px to 8px to 4px; grays shift from `#666` to `#555` to `#777`; spacing varies from 12px to 16px to 14px. Individually trivial, collectively these inconsistencies make a platform feel unprofessional. The design system prevents this by encoding every visual decision as a token, and every token is consumed via Tailwind classes.
    16→
    17→This document is enforced via Tailwind configuration (tokens mapped to Tailwind theme), shadcn/ui component library (consistent primitives), ESLint rules (no inline styles, no hardcoded colors), and code review. Per EC-10, no page may introduce ad hoc UI patterns — every visual element originates from this system.

    40→## 3. Architectural Decisions
    41→
    42→### AD-01 — Token-Driven Theming via CSS Custom Properties
    43→
    44→**Context.** Hardcoded colors in components break dark mode and prevent theming. CSS-in-JS solutions add runtime overhead. A token-driven approach using CSS custom properties enables theme switching without re-renders.
    45→
    46→**Decision.** Design tokens are defined as CSS custom properties in `globals.css` at the `:root` level (light theme) and `.dark` class (dark theme). Tailwind theme extension maps these tokens to Tailwind classes (e.g., `bg-background` → `background-color: var(--background)`).
    47→
    48→```css
    49→:root {
    50→  --background: 0 0% 100%;           /* white */
    51→  --foreground: 0 0% 3.9%;           /* near black */
    52→  --primary: 0 0% 9%;                /* primary action */
    53→  --primary-foreground: 0 0% 98%;
    54→  /* ... more tokens */
    55→}
    56→
    57→.dark {
    58→  --background: 0 0% 3.9%;
    59→  --foreground: 0 0% 98%;
    60→  --primary: 0 0% 98%;
    61→  --primary-foreground: 0 0% 9%;
    62→  /* ... more tokens */
    63→}
    64→```
    65→
    66→Tailwind config:
    67→```typescript
    68→module.exports = {
    69→  theme: {
    70→    extend: {
    71→      colors: {
    72→        background: 'hsl(var(--background))',
    73→        foreground: 'hsl(var(--foreground))',
    74→        primary: { DEFAULT: 'hsl(var(--primary))', foreground: 'hsl(var(--primary-foreground))' },
    75→        // ... more
    76→      },
    77→    },
    78→  },
    79→};
    80→```
    81→
    82→**Consequences.**
    83→- ✅ Theme switching via class toggle on `<html>` — no re-renders.
    84→- ✅ Consistent token usage across all components.
    85→- ✅ Dark mode supported from day 1 (LOCK-10).
    86→- ⚠️ Slight indirection (token → CSS var → Tailwind class); justified by flexibility.
    87→
    88→**Implements:** LOCK-10 (Dark/Light mode ready), EC-10 (Design System Governance).
    89→
    90→### AD-02 — Monochrome Palette with Single Accent
    91→
    92→**Context.** LOCK-10 mandates developer-first minimalism with a monochrome palette. Adding multiple accent colors fragments the visual language.
    93→
    94→**Decision.** Palette:
    95→- **Background and foreground:** pure grayscale (white to near-black).
    96→- **Primary actions:** near-black in light mode, near-white in dark mode (inverted).
    97→- **Accent color:** single accent reserved for highlighting (links, focused elements, badges). The accent is configurable via token; default is a subtle blue (`hsl(221, 83%, 53%)`) for familiarity, but can be re-themed without code changes.
    98→- **Semantic colors:** success (green), warning (yellow), destructive (red), info (blue). Used sparingly for status indicators, not decoration.
    99→
   100→All other colors are grayscale. No gradients, no decorative color.
   101→
   102→**Consequences.**
   103→- ✅ Consistent, professional aesthetic.
   104→- ✅ High contrast (WCAG AAA where feasible — EC-06).
   105→- ✅ Color-blind friendly (semantic meaning conveyed via color + icon, not color alone).
   106→
   107→**Implements:** LOCK-10, EC-06.
   108→
   109→### AD-03 — Typography Scale
   110→
   111→**Context.** Without a defined type scale, font sizes drift (14px, 15px, 16px, 17px all appearing). A modular scale ensures hierarchy and rhythm.
   112→
   113→**Decision.** Typography scale (rem-based, 1rem = 16px):
   114→
   115→| Token | Size | Weight | Use |
   116→|-------|------|--------|-----|
   117→| `text-xs` | 0.75rem (12px) | 400 | Captions, badges, small print |
   118→| `text-sm` | 0.875rem (14px) | 400 | Secondary text, labels, table cells |
   119→| `text-base` | 1rem (16px) | 400 | Body text, default |
   120→| `text-lg` | 1.125rem (18px) | 500 | Section headings, emphasized text |
   121→| `text-xl` | 1.25rem (20px) | 600 | Page subheadings |
   122→| `text-2xl` | 1.5rem (24px) | 600 | Page headings |
   123→| `text-3xl` | 1.875rem (30px) | 700 | Hero headings |
   124→| `text-4xl` | 2.25rem (36px) | 700 | Large hero, marketing |
   125→| `text-5xl` | 3rem (48px) | 700 | Marketing only |
   126→
   127→**Font families:**
   128→- **Sans-serif (default):** `Inter` (loaded via `next/font/google`). Used for UI, body text.
   129→- **Mono:** `JetBrains Mono` (loaded via `next/font/google`). Used for code blocks, technical identifiers.
   130→- **No serif fonts** in Phase 1. May add for editorial content (articles) in Phase 2.
   131→
   132→**Line heights:**
   133→- Body: 1.5
   134→- Headings: 1.2
   135→- Code: 1.6
   136→
   137→**Letter spacing:**
   138→- Body: normal (0)
   139→- Headings: -0.01em (slightly tighter for large text)
   140→- Small text: 0.01em (slightly looser for readability)
   141→
   142→**Implements:** LOCK-10 (premium feel via precise typography).
   143→
   144→### AD-04 — Spacing Scale
   145→
   146→**Context.** Inconsistent spacing (12px, 14px, 16px, 18px padding) breaks visual rhythm. Tailwind's spacing scale (4px base) provides consistency.
   147→
   148→**Decision.** Use Tailwind's default spacing scale:
   149→
   150→| Token | Size | Use |
   151→|-------|------|-----|
   152→| `0` | 0 | No spacing |
   153→| `1` | 4px | Tight inline spacing |
   154→| `2` | 8px | Compact element padding |
   155→| `3` | 12px | Small element padding |
   156→| `4` | 16px | Default element padding |
   157→| `6` | 24px | Section internal spacing |
   158→| `8` | 32px | Section spacing |
   159→| `12` | 48px | Page section spacing |
   160→| `16` | 64px | Large page section spacing |
   161→| `24` | 96px | Hero spacing |
   162→
   163→**Rules:**
   164→- Use only these values; no `p-[13px]` arbitrary values.
   165→- Spacing between related elements: `2` or `3` (8-12px).
   166→- Spacing within a section: `4` or `6` (16-24px).
   167→- Spacing between sections: `8` or `12` (32-48px).
   168→- Page-level rhythm: multiples of 8px.
   169→
   170→**Implements:** LOCK-10 (spacious layout), EC-10 (consistent spacing).
   171→
   172→### AD-05 — Border Radius Scale
   173→
   174→**Context.** Inconsistent corner radii (4px, 6px, 8px, 12px) look unprofessional.
   175→
   176→**Decision.** Radius scale:
   177→
   178→| Token | Radius | Use |
   179→|-------|--------|-----|
   180→| `rounded-none` | 0 | Tables, dividers |
   181→| `rounded-sm` | 2px | Small inline elements, badges |
   182→| `rounded-md` | 6px | Inputs, buttons (default) |
   183→| `rounded-lg` | 8px | Cards, modals |
   184→| `rounded-xl` | 12px | Large cards, feature panels |
   185→| `rounded-full` | 9999px | Pills, avatars, circular buttons |
   186→
   187→**Rules:**
   188→- Default radius: `rounded-md` (6px).
   189→- Cards: `rounded-lg` (8px) or `rounded-xl` (12px) for emphasis.
   190→- Inputs and buttons: `rounded-md` (6px).
   191→- Never mix radii in a single component (e.g., button with `rounded-lg` inside a `rounded-md` card).
   192→
   193→**Implements:** LOCK-10 (consistent component usage).
   194→
   195→### AD-06 — Shadow Scale
   196→
   197→**Context.** Shadows convey depth. Without a scale, shadows vary wildly.
   198→
   199→**Decision.** Shadow scale (subtle, developer-first aesthetic):
   200→
   201→| Token | Shadow | Use |
   202→|-------|--------|-----|
   203→| `shadow-none` | none | Flat elements |
   204→| `shadow-sm` | `0 1px 2px 0 rgb(0 0 0 / 0.05)` | Subtle elevation (inputs on focus) |
   205→| `shadow-md` | `0 4px 6px -1px rgb(0 0 0 / 0.1), 0 2px 4px -2px rgb(0 0 0 / 0.1)` | Cards, dropdowns |
   206→| `shadow-lg` | `0 10px 15px -3px rgb(0 0 0 / 0.1), 0 4px 6px -4px rgb(0 0 0 / 0.1)` | Modals, popovers |
   207→| `shadow-xl` | `0 20px 25px -5px rgb(0 0 0 / 0.1), 0 8px 10px -6px rgb(0 0 0 / 0.1)` | Floating panels |
   208→
   209→**Rules:**
   210→- Default: `shadow-none` (flat aesthetic).
   211→- Use `shadow-sm` for inputs on focus.
   212→- Use `shadow-md` for cards with hover interaction.
   213→- Use `shadow-lg` for modals and popovers.
   214→- Dark mode: shadows are less visible; rely more on border for depth.
   215→
   216→**Implements:** LOCK-10 (minimal animation, premium feel via precise elevation).
   217→
   218→### AD-07 — Motion and Animation Standards
   219→
   220→**Context.** LOCK-10 mandates minimal animation. Excessive motion degrades the developer-first aesthetic and can cause accessibility issues (motion sensitivity).
   221→
   222→**Decision.** Motion standards:
   223→
   224→**Allowed animations (state-change only):**
   225→- Hover transitions: 150ms ease-out.
   226→- Focus transitions: 100ms ease-out.
   227→- Modal/dialog open: 200ms ease-out.
   228→- Modal/dialog close: 150ms ease-in.
   229→- Toast notifications: 200ms ease-out.
   230→
   231→**Forbidden animations:**
   232→- Decorative animations (pulsing, bouncing, rotating for non-loading states).
   233→- Auto-playing carousels.
   234→- Parallax scrolling.
   235→- Animated illustrations.
   236→- Page transition animations (instant page changes preferred).
   237→
   238→**Loading states:**
   239→- Skeleton loaders with subtle pulse (1.5s ease-in-out infinite).
   240→- Progress bars for long operations.
   241→- No spinner-only loading states (always skeleton or progress).
   242→
   243→**Reduced motion:**
   244→- All animations respect `prefers-reduced-motion: reduce` (EC-06).
   245→- When reduced motion is preferred, animations are instant or replaced with opacity changes.
   246→
   247→```css
   248→@media (prefers-reduced-motion: reduce) {
   249→  *, *::before, *::after {
   250→    animation-duration: 0.01ms !important;
   251→    transition-duration: 0.01ms !important;
   252→  }
   253→}
   254→```
   255→
   256→**Implements:** LOCK-10 (minimal animation), EC-06 (reduced motion compatibility).
   257→
   258→### AD-08 — Layout Grid and Breakpoints
   259→
   260→**Context.** Responsive design requires consistent breakpoints. Tailwind's default breakpoints are well-established.
   261→
   262→**Decision.** Breakpoints (Tailwind defaults):
   263→
   264→| Prefix | Min Width | Use |
   265→|--------|-----------|-----|
   266→| (default) | 0 | Mobile-first baseline (360px+) |
   267→| `sm:` | 640px | Large phones, small tablets |
   268→| `md:` | 768px | Tablets |
   269→| `lg:` | 1024px | Small laptops |
   270→| `xl:` | 1280px | Desktops |
   271→| `2xl:` | 1536px | Large desktops |
   272→
   273→**Container:**
   274→- Max width: `max-w-7xl` (1280px) for most pages.
   275→- Max width: `max-w-4xl` (896px) for article pages (reading width).
   276→- Padding: `px-4` on mobile, `px-6` on `sm:`, `px-8` on `lg:`.
   277→- Centered: `mx-auto`.
   278→
   279→**Grid:**
   280→- Use Tailwind's grid utilities (`grid`, `grid-cols-*`, `gap-*`).
   281→- Tool grid: `grid-cols-1 sm:grid-cols-2 md:grid-cols-3 lg:grid-cols-4`.
   282→- Article grid: `grid-cols-1 md:grid-cols-2 lg:grid-cols-3`.
   283→
   284→**Mobile-first:**
   285→- All components designed for 360px viewport first.
   286→- Use `min-w-*` for minimum sizes, not `max-w-*` for mobile.
   287→- Touch targets minimum 44x44px (WCAG recommendation).
   288→
   289→**Implements:** LOCK-10 (mobile-first, responsive), EC-06 (accessibility — touch targets).
   290→
   291→### AD-09 — Icon System
   292→
   293→**Context.** Mixing icon sources (SVG sprites, icon fonts, custom SVGs) creates inconsistency.
   294→
   295→**Decision.** Icon system:
   296→- **Library:** `lucide-react` (consistent with shadcn/ui).
   297→- **Size:** 16px (default), 20px (medium), 24px (large). Use Tailwind `w-4 h-4`, `w-5 h-5`, `w-6 h-6`.
   298→- **Color:** inherits from `currentColor` (use `text-foreground`, `text-muted-foreground`, etc.).
   299→- **Stroke width:** 2px (default).
   300→- **No custom SVGs** unless the icon is not available in lucide-react. Document exceptions in component.
   301→- **No icon fonts.**
   302→
   303→**Implements:** EC-10 (Design System Governance — consistent icon source).
   304→
   305→### AD-10 — Accessibility Standards
   306→
   307→**Context.** EC-06 mandates WCAG AA conformance. Specific standards make this enforceable.
   308→
   309→**Decision.** Accessibility standards:
   310→
   311→**Color contrast:**
   312→- Body text on background: ≥7:1 (WCAG AAA).
   313→- Large text (≥24px or ≥18.66px bold) on background: ≥4.5:1 (WCAG AA).
   314→- UI components and graphical elements: ≥3:1.
   315→- Verified via automated tooling (Lighthouse CI) and manual review.
   316→
   317→**Keyboard navigation:**
   318→- All interactive elements operable via keyboard.
   319→- Logical tab order (follows visual order).
   320→- Visible focus indicator (`focus-visible:` Tailwind classes with `outline-2 outline-offset-2 outline-primary`).
   321→- No keyboard traps.
   322→- Skip-to-content link at top of every page.
   323→
   324→**Screen reader support:**
   325→- Semantic HTML (`<button>`, `<nav>`, `<main>`, `<article>`, `<section>`, `<aside>`, `<header>`, `<footer>`).
   326→- ARIA labels only when semantic HTML insufficient.
   327→- `aria-label` on icon-only buttons.
   328→- `aria-live="polite"` on dynamic content (toasts, alerts).
   329→- `aria-busy` on loading elements.
   330→- Hidden content via `aria-hidden="true"` (not `display: none` if it should be screen-reader accessible).
   331→
   332→**Forms:**
   333→- Every input has associated `<label>`.
   334→- Error messages via `aria-describedby` linking to input.
   335→- Required fields marked with `aria-required="true"` and visible indicator.
   336→- Validation errors announced via `aria-live`.
   337→
   338→**Reduced motion:**
   339→- Per AD-07. `prefers-reduced-motion` respected.
   340→
   341→**Touch targets:**
   342→- Minimum 44x44px on touch devices.
   343→- Adequate spacing between interactive elements.
   344→
   345→**Images:**
   346→- All images have `alt` attribute.
   347→- Decorative images: `alt=""` (empty).
   348→- Complex images: `aria-describedby` linking to longer description.
   349→
   350→**Implements:** EC-06 (Accessibility First).
   351→
   352→### AD-11 — Dark Mode Strategy
   353→
   354→**Context.** LOCK-10 mandates dark mode ready from day 1. The strategy must be token-driven (no hardcoded colors) and toggleable.
   355→
   356→**Decision.** Dark mode strategy:
   357→- **Token-driven:** All colors via CSS custom properties (AD-01).
   358→- **Class-based theming:** `.dark` class on `<html>` element toggles dark mode.
   359→- **User preference:** Stored in cookie (server-readable for SSR) and localStorage.
   360→- **System preference:** Default to `prefers-color-scheme` if no user preference.
   361→- **No flash of incorrect theme:** Server-rendered `<html>` class based on cookie.
   362→
   363→**Implementation:**
   364→```typescript
   365→// src/app/layout.tsx
   366→export async function generateMetadata() {
   367→  // ...
   368→}
   369→
   370→export default function RootLayout({ children }: { children: React.ReactNode }) {
   371→  return (
   372→    <html lang="en" suppressHydrationWarning>
   373→      <head>
   374→        <script
   375→          dangerouslySetInnerHTML={{
   376→            __html: `
   377→              (function() {
   378→                const theme = document.cookie.match(/theme=(dark|light)/)?.[1]
   379→                  || (window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light');
   380→                document.documentElement.classList.toggle('dark', theme === 'dark');
   381→              })();
   382→            `,
   383→          }}
   384→        />
   385→      </head>
   386→      <body>{children}</body>
   387→    </html>
   388→  );
   389→}
   390→```
   391→
   392→**Implements:** LOCK-10 (Dark/Light mode ready).
   393→
   394→### AD-12 — Z-Index Scale
   395→
   396→**Context.** Without a z-index scale, values drift and stacking becomes unpredictable.
   397→
   398→**Decision.** Z-index scale:
   399→
   400→| Token | Value | Use |
   401→|-------|-------|-----|
   402→| `z-0` | 0 | Default |
   403→| `z-10` | 10 | Sticky headers, dropdowns |
   404→| `z-20` | 20 | Sticky sidebars |
   405→| `z-30` | 30 | Modals |
   406→| `z-40` | 40 | Popovers, tooltips |
   407→| `z-50` | 50 | Toast notifications |
   408→| `z-[100]` | 100 | Critical overlays (error boundaries) |
   409→
   410→**Rules:**
   411→- Use only these values.
   412→- Never use `z-[9999]` or other arbitrary values.
   413→- Higher z-index = more "on top"; respect the hierarchy.
   414→
   415→**Implements:** EC-10 (consistent stacking).

   417→## 4. Design Principles
   418→
   419→### P1 — Token-Driven, Always
   420→Every color, spacing, radius, shadow, font size comes from a token. No hardcoded values. This is the foundation of dark mode and theming.
   421→
   422→### P2 — Minimal by Default
   423→Start with the minimum viable visual treatment. Add decoration only when it improves usability. LOCK-10 mandates this.
   424→
   425→### P3 — Hierarchy Through Contrast, Not Decoration
   426→Visual hierarchy via typography size/weight, spacing, and color contrast — not via decorative elements (borders, gradients, shadows).
   427→
   428→### P4 — Accessible by Construction
   429→Accessibility is built into tokens (contrast ratios) and components (semantic HTML, ARIA). Not bolted on later.
   430→
   431→### P5 — Consistent Across the Ecosystem
   432→Every tool page, every admin page, every marketing page uses the same tokens and components. The user feels they're in one product, not a patchwork.
   433→
   434→### P6 — Performance-Conscious
   435→Minimal CSS, no runtime CSS-in-JS, fonts loaded via `next/font` for optimization. Design choices must not bloat the bundle.

   721→## 8. Standards
   722→
   723→### 8.1 Token Usage Standards
   724→- All colors via tokens (`bg-background`, `text-foreground`, etc.).
   725→- No hardcoded hex values.
   726→- No `style={{ color: '#fff' }}` inline styles.
   727→
   728→### 8.2 Component Usage Standards
   729→- Use design system components from `@packages/ui`.
   730→- Compose into higher-level components in `src/shared/components/`.
   731→- Don't reinvent primitives (Button, Input, Card, etc.).
   732→
   733→### 8.3 Dark Mode Standards
   734→- Every component works in light and dark mode.
   735→- Test both modes during development.
   736→- No dark-mode-specific code paths; token switching handles it.
   737→
   738→### 8.4 Accessibility Standards
   739→- Per AD-10. WCAG AA conformance mandatory.
   740→- Verified via Lighthouse CI (≥95 accessibility score).
   741→
   742→### 8.5 Animation Standards
   743→- Per AD-07. Minimal animation.
   744→- Reduced motion respected.
   745→
   746→### 8.6 Performance Standards
   747→- No runtime CSS-in-JS.
   748→- Fonts via `next/font`.
   749→- Minimal custom CSS (only `globals.css`).
   750→
   751→## 9. Best Practices
   752→
   753→### 9.1 When Choosing Colors
   754→1. Use tokens, never hex values.
   755→2. Default to muted/foreground for text.
   756→3. Use primary for actions, accent for highlights.
   757→4. Use semantic colors (success/warning/destructive) only for status.
   758→5. Verify contrast in both light and dark mode.
   759→
   760→### 9.2 When Spacing Elements
   761→1. Use Tailwind spacing scale (4, 8, 12, 16, 24, 32, 48, 64).
   762→2. Within a section: 16-24px (`gap-4` or `gap-6`).
   763→3. Between sections: 32-48px (`py-8` or `py-12`).
   764→4. Page padding: 16px mobile, 24-32px desktop.
   765→
   766→### 9.3 When Adding a Component
   767→1. Check `@packages/ui` for existing (EC-03).
   768→2. Check `src/shared/components/` for app-specific composite.
   769→3. If new: place in appropriate location per `07_FolderStructure`.
   770→4. Use design tokens, not hardcoded values.
   771→5. Verify dark mode and accessibility.
   772→
   773→### 9.4 When Animating
   774→1. Default to no animation.
   775→2. Add animation only for state changes (hover, focus, open/close).
   776→3. Keep duration ≤200ms.
   777→4. Respect `prefers-reduced-motion`.
   778→5. No decorative animations.
   779→
   780→### 9.5 When Theming
   781→1. All theming via tokens.
   782→2. To change theme: modify CSS custom properties in `globals.css`.
   783→3. Dark mode: toggle `.dark` class on `<html>`.
   784→4. Test both modes.
